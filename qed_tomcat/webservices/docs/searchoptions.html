queryFeatures OPTION GROUP
Option Type Description
implicitHMatching STRING default: DEFAULT
  • DEFAULT : Default enables matching in most cases, but in case of duplicate search the default value is disabled
  • ENABLED : The number of implicit hydrogens is checked
  • DISABLED : The number of implicit hydrogens is not checked
  • IGNORE : Implicit hydrogens are ignored during searching
undefinedRAtom STRING default: GROUP
  • GROUP : Group is default, undefined R-atom in query matches a group of one or more connected atoms, including at least one heavy atom
  • GROUP_H : Undefined R-atom in query matches a group of one or more connected atoms, which can also be a single H atom
  • GROUP_H_EMPTY : Same as GROUP_H plus the target atoms can be the empty set (empty set match is allowed for isolated or one-attachment R-atoms only)
  • ANY : Undefined R-atom matches any single atom in target
  • UNDEFINED : Undefined R-atom matches only an undefined R atom in target
bridgingRLigand BOOLEAN default: false
False is default, different R-atoms are not allowed to match the same group of atoms, true allows different R-atoms to match the same group of target atoms
rLigandEqualityCheck BOOLEAN default: true
True is default, it switches on the requirement that R-atoms with the same R-group ID should match ligands with the same structure. False enables switching off the requirement that R-atoms with the same R-group ID should match ligands with the same structure
attachedDataMatch STRING default: IGNORE
  • IGNORE : Ignore is default, attached data is ignored in search
  • GENERAL : If there is attached data defined in query, attached data fields in the query and target with the same name are compared
  • EXACT : In exact mode both operators and values must match (an attached data existing only in target prevents matching)
exactQueryAtomMatching BOOLEAN default: false
False is default, exact query atom matching should not be used. True effects query information in atoms not to be evaluated, but rather exact matching of these information is required
attachedDataPrefixes STRING This value is used only when attached data is not ignored. Sets the prefixes of the names of those data sgroups that will be used during structure comparison
